• Bank-Account-App is an application base microservices with business microservices: customer, bank-account, operation, notification
  • each business microservice has its own db
  • each business microservice implements a architecture hexagonale
  • business microservices interact between: bank-account interacts with customer and operation interacts with bank-account
  • gateway-service-proxy routes frontend app requests to the bankend microservices
  • Keycloak identity provider authenticate and authorize all users' requests
  • Bank-Account-App stack is deployed into docker images
  • on backend side, kafka infrastructure is used to publish events related to users requests
  • bs-ms-notification sends a notification email on smtp-email-server when an event is published in kafka topic
  • docker containers of Bank-Account-App stack are deployed into a minikube cluster using Kubernetes
  • for more details: git repo
